## 一、先看框架，再抠细节

你的误区可能是：打开 `camera.c` 就开始一行行读，读了半小时不知道这段代码在整个项目里什么位置。

**正确顺序：先理解项目架构，再拆到文件，最后看具体代码。**

### 第0步（30分钟）：理解整体架构

打开三个文件，只看函数名和调用关系：

```
main.c（入口）
├── nvs_flash_init()          → 初始化存储
├── wifi_init()               → 连 WiFi
├── camera_init_custom()      → 初始化摄像头
│   └── esp_camera_init()     → 底层驱动
├── http_server_start()       → 启动 HTTP 服务
│   └── video_handler()       → MJPEG 推流
└── mqtt_app_start()          → 启动 MQTT
    ├── mqtt_event_handler()  → 处理消息
    └── video_stream_task()   → 推流任务
```

**看懂这张图，你就知道整个项目是怎么串起来的了。** 面试官问"项目整体架构"，你就按这个顺序说：先 NVS → WiFi → 摄像头 → HTTP 服务 → MQTT。

## 二、5 个文件逐一看，每个文件分三层

每个文件都按这个节奏，总共约 5 天，每晚 2 个番茄：

```
番茄①（25分钟）：打开文件通读一遍
              → 只看函数名，不读每一行
              → 问自己：这个文件暴露了几个函数？给谁用？
番茄②（25分钟）：关掉文件，用嘴说
              → 这个文件做了什么？
              → 用到了什么技术？
              → 能说出 3 句以上就算过关
```

### 文件 1：`camera.c`（1 个番茄）

```
看什么：
  → camera_init_custom() 函数
  → camera_config_t 结构体配置：引脚、XCLK、分辨率、JPEG、双缓冲
  → sensor_t 指针设置分辨率

关掉代码后能说出：
  "我对着 AI-Thinker 的原理图配了 OV2640 的 16 个引脚，
   XCLK 降到 10MHz 解决花屏，VGA+JPEG输出，
   双缓冲+LATEST 模式让图像延迟更低。"
```

涉及八股：
- `结构体赋值语法`（C语言）
- `I2C/SCCB 协议`（OV2640 控制接口）
- `JPEG 编码概念`（为什么用 JPEG 不用 BMP）

### 文件 2：`wifi.c`（1 个番茄）

```
看什么：
  → wifi_event_handler() 回调：STA_START → 连接 / DISCONNECTED → 重试
  → EventGroup 创建和等待

关掉代码后能说出：
  "用的 STA 模式，注册了 WIFI_EVENT 和 IP_EVENT 回调，
   断网后在 DISCONNECTED 事件里重试最多 5 次，
   用 EventGroup 的 CONNECTED_BIT 和 FAIL_BIT 同步连接状态。"
```

涉及八股：
- `事件驱动编程`（回调函数概念）
- `FreeRTOS 事件标志组`（EventGroup API）
- `三次握手/TCP 基础`（WiFi 连接底层）

### 文件 3：`http_server.c`（1 个番茄）

```
看什么：
  → video_handler()：设置响应类型为 multipart/x-mixed-replace
  → 每一帧怎么封装：--frame 边界 + JPEG 头 + 数据 + 帧尾
  → JPEG 校验：0xFF 0xD8 0xFF

关掉代码后能说出：
  "用 ESP-IDF 的 HTTP Server 库，8080 端口，
   响应类型设成 multipart/x-mixed-replace 浏览器就能直接显示视频流，
   每帧检查前三个字节是不是 JPEG 文件头，不是就丢掉。"
```

涉及八股：
- `HTTP 协议基础`（请求/响应、Content-Type）
- `MJPEG 格式原理`（每帧独立 JPEG，无帧间压缩）
- `内存管理`（帧缓冲及时释放，防止泄漏）

### 文件 4：`mqtt.c`（1 个番茄）

```
看什么：
  → mqtt_event_handler()：CONNECTED/DISCONNECTED/DATA 事件
  → 订阅主题 camera/control

关掉代码后能说出：
  "MQTT 客户端连 EMQX 代理，订阅 camera/control 主题，
   连接成功就发订阅，收到消息根据内容执行指令。"
```

涉及八股：
- `MQTT 协议`（发布/订阅模型、QoS、主题）
- `TCP/IP 通信基础`
- `FreeRTOS 任务创建`（xTaskCreate 参数含义）

### 文件 5：`main.c`（0.5 个番茄）

```
看什么：
  → app_main() 启动顺序
  → Flash 容量诊断逻辑

关掉代码后能说出：
  "启动顺序：NVS → WiFi → 摄像头 → HTTP → MQTT。
   Flash 诊断用 esp_flash_get_physical_size 获取物理容量，
   配合 esp_partition_find 遍历分区表算已用和剩余。"
```

涉及八股：
- `ESP-IDF 分区表结构`（nvs / phy_init / factory）
- `Flash 存储基础`（Nor Flash 读写特性）

## 三、什么叫"过关"——三条标准

每条简历上的描述，你要能接住对应的追问：

| 简历写了 | 面试官追问 | 你能答出来就算过关 |
|---------|----------|-----------------|
| XCLK 降频至 10MHz | 为什么降？ | "20MHz 花屏→降到10MHz稳定了" |
| 双缓冲+LATEST | 双缓冲原理？ | "两个 buf 交替用，一个采集一个处理" |
| STA 模式 5 次重连 | 重试在哪实现的？ | "wifi_event_handler 的 DISCONNECTED 里" |
| EventGroup 同步 | EventGroup 干嘛的？ | "存两个 bit，主任务 wait 直到有结果" |
| MJPEG 视频流 | 怎么在浏览器显示？ | "multipart/x-mixed-replace 格式，每帧"--frame"分割" |
| JPEG 校验 | 怎么校验的？ | "检查前三个字节 0xFF 0xD8 0xFF" |
| 分区表扩容 | 怎么扩的？ | "partitions.csv 里 factory 从 1MB 改 1.5MB" |
| PSRAM 配置 | 在哪配的？ | "menuconfig 里开 SPI RAM 支持" |

**8 个问题，你每个能答出 2 句话就算过关。** 不需要长篇大论。
